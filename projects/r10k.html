R10K Style Out-of-Order Processor

Design Overview

Our team implemented the R10000 architecture, selecting it to reduce implementation risks due to a strong grasp of its structural logic[cite: 3, 4]. We engineered an N-way scaled out-of-order processor featuring advanced GShare branch prediction, prefetching with an unblocked instruction cache, and an associative data cache[cite: 5].

Component Specifications

  • Reservation Station (RS): 8 entries[cite: 66].
  • Reorder Buffer (ROB): 15 entries[cite: 67].
  • Physical Registers (PR): 64 registers[cite: 68].
  • Functional Units: 2 ALU, 2 Load, 2 Stores, 2 Mult[cite: 68].
  • Instruction Cache: 256 bytes[cite: 74].
  • Prefetcher: Looks 3 lines ahead[cite: 75].
  • Store Queue (SQ): 8 entries[cite: 75].
  • Data Cache: 256 bytes[cite: 76].
  • PHT / BTB: 16 entries each[cite: 76].

Pipeline Architecture

  • Fetch Stage: Retrieves up to two instructions per cycle[cite: 80]. It requests 64-bit aligned blocks from the ICache using dynamic alignment logic [cite: 84] and monitors the Instruction Queue to gracefully transition to a stall state upon detecting backpressure[cite: 86].
  • Instruction Queue: A 2N-depth circular FIFO buffer [cite: 89] supporting simultaneous read/writes[cite: 90]. It uses a lookahead mechanism to pre-calculate dispatchable packets[cite: 91].
  • Dispatch Stage: Parallelizes decoding, register renaming, and resource arbitration[cite: 95]. Maps destination registers to physical tags from the Free List [cite: 98] and assigns Store Queue indices[cite: 99, 100].
  • Reservation Station: Enables out-of-order issues by holding decoded instructions until dependencies are met[cite: 102]. Accepts up to N new instructions [cite: 103] and uses a parallel ready-mask against CDB broadcasts for instant wakeup[cite: 104].
  • Reorder Buffer (ROB): Maintains precise state with a circular buffer supporting superscalar advancement[cite: 110]. Ensures in-order retirement [cite: 113] and acts as the central recovery unit during branch mispredictions[cite: 114].
  • Free List & Map Table: The Free List exposes a vector of N candidate registers [cite: 119], while the Map Table handles dynamic architectural-to-physical mapping [cite: 126] and instantly restores state from the committed architectural map during a flush[cite: 129].

Memory Interface

  • Non-Blocking Instruction Cache: Utilizes dual memory ports [cite: 132] to stream 4 instructions simultaneously, enabling a 3-instruction lookahead[cite: 133].
  • Data Cache: A 4-way set-associative, write-allocate, write-back design [cite: 137] managed through a controller FSM and a single-entry MSHR[cite: 138].
  • Memory Controller: Manages backpressure by strictly prioritizing store commits over load requests[cite: 143].

Advanced Features & Performance

  • N-way Scalar Execution: Fully parameterized pipeline processing up to N instructions per cycle [cite: 149], supported by N-ported backend structures[cite: 152]. Optimal performance was achieved at N=3 or N=4[cite: 374].
  • Stream Buffer Prefetching: A non-blocking prefetcher scans a four-line lookahead window[cite: 154, 155]. This single addition yielded a ~1.8x speedup on core benchmarks (improving CPI from 9 to 5)[cite: 357].
  • GShare Branch Predictor: Integrates a 4-bit branch history register [cite: 159], utilizing an XOR hash of the PC and history to index the Pattern History Table [cite: 163], alongside a tagged Branch Target Buffer to prevent aliasing[cite: 164].
