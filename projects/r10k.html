R10K Style Out-of-Order Processor

SystemVerilog • Synopsys Design Compiler • Synopsys Verdi • CPU Microarchitecture

An N-way scaled out-of-order processor based on the R10000 architecture, featuring GShare branch prediction, stream buffer prefetching, and an associative data cache.

Top-level block diagram of the N-way superscalar pipeline.

Architecture at a Glance

  • Fetch/Dispatch: Up to N instructions/cycle
  • Reservation Station: 8 entries
  • Reorder Buffer (ROB): 15 entries
  • Physical Registers: 64 registers
  • Caches: 256B I-Cache (Non-blocking) | 256B D-Cache (4-way associative)

Pipeline Walkthrough

Following the data path illustrated above, the processor manages out-of-order execution through the following highly-parameterized stages:

1. Front-End: Fetch & Predict

The pipeline begins with a dual-port, non-blocking Instruction Cache that streams up to 4 instructions simultaneously. A stream buffer prefetcher looks 3 lines ahead, which alone yielded a ~1.8x speedup on core benchmarks.

2. Dispatch & Renaming

To support N-way execution, the Dispatch stage parallelizes decoding and resource arbitration. The Map Table handles dynamic architectural-to-physical mapping, pulling from a Free List that exposes a vector of N candidate registers. If the ROB, Store Queue, or Free List hits capacity, the instruction queue gracefully stalls.

3. Out-of-Order Execution

Instructions are held in the 8-entry Reservation Station until dependencies are cleared. A parallel ready-mask constantly snoops the Common Data Bus (CDB) broadcasts for instant wakeup. Once ready, instructions fire into one of the available functional units (2 ALU, 2 Load, 2 Store, 2 Mult).

4. Memory Subsystem

The Data Cache is a 4-way set-associative, write-allocate, write-back design. To manage complex memory dependencies and prevent pipeline lockups, a dedicated Memory Controller strictly prioritizes store commits over load requests, utilizing a single-entry MSHR.

5. In-Order Retirement

The 15-entry Reorder Buffer (ROB) is the heart of the processor's precise state. Operating as a circular buffer, it supports superscalar advancement and acts as the central recovery unit. On a branch misprediction, the ROB flushes speculative states and instantly restores the correct mapping from the committed architectural map.

Results & Scalability

Because the backend structures (like the Map Table and Free List) are fully N-ported, the processor scales dynamically. During performance testing, optimal throughput was achieved at N=3 or N=4, effectively reducing the CPI from 9 to 5 on critical workloads.
