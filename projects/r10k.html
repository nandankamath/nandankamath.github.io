R10K Style Out-of-Order Processor

Design Overview

Our team implemented the R10000 architecture, selecting it to reduce implementation risks due to a strong grasp of its structural logic. We engineered an N-way scaled out-of-order processor featuring advanced GShare branch prediction, prefetching with an unblocked instruction cache, and an associative data cache.

Component Specifications

  • Reservation Station (RS): 8 entries.
  • Reorder Buffer (ROB): 15 entries.
  • Physical Registers (PR): 64 registers.
  • Functional Units: 2 ALU, 2 Load, 2 Stores, 2 Mult.
  • Instruction Cache: 256 bytes.
  • Prefetcher: Looks 3 lines ahead.
  • Store Queue (SQ): 8 entries.
  • Data Cache: 256 bytes.
  • PHT / BTB: 16 entries each.

Pipeline Architecture

  • Fetch Stage: Retrieves up to two instructions per cycle. It requests 64-bit aligned blocks from the ICache using dynamic alignment logic and monitors the Instruction Queue to gracefully transition to a stall state upon detecting backpressure.
  • Instruction Queue: A 2N-depth circular FIFO buffer supporting simultaneous read/writes. It uses a lookahead mechanism to pre-calculate dispatchable packets.
  • Dispatch Stage: Parallelizes decoding, register renaming, and resource arbitration. Maps destination registers to physical tags from the Free List and assigns Store Queue indices.
  • Reservation Station: Enables out-of-order issues by holding decoded instructions until dependencies are met. Accepts up to N new instructions and uses a parallel ready-mask against CDB broadcasts for instant wakeup.
  • Reorder Buffer (ROB): Maintains precise state with a circular buffer supporting superscalar advancement. Ensures in-order retirement and acts as the central recovery unit during branch mispredictions.
  • Free List & Map Table: The Free List exposes a vector of N candidate registers, while the Map Table handles dynamic architectural-to-physical mapping and instantly restores state from the committed architectural map during a flush.

Memory Interface

  • Non-Blocking Instruction Cache: Utilizes dual memory ports to stream 4 instructions simultaneously, enabling a 3-instruction lookahead.
  • Data Cache: A 4-way set-associative, write-allocate, write-back design managed through a controller FSM and a single-entry MSHR.
  • Memory Controller: Manages backpressure by strictly prioritizing store commits over load requests.

Advanced Features & Performance

  • N-way Scalar Execution: Fully parameterized pipeline processing up to N instructions per cycle, supported by N-ported backend structures. Optimal performance was achieved at N=3 or N=4.
  • Stream Buffer Prefetching: A non-blocking prefetcher scans a four-line lookahead window. This single addition yielded a ~1.8x speedup on core benchmarks (improving CPI from 9 to 5).
  • GShare Branch Predictor: Integrates a 4-bit branch history register, utilizing an XOR hash of the PC and history to index the Pattern History Table, alongside a tagged Branch Target Buffer to prevent aliasing.
